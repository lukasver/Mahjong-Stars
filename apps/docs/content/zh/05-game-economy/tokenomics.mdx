### 分配理念

**以玩家为中心的分配**:
- **游戏内奖励（33.33%）**: 最大单一分配，确保长期的玩家激励。
- **综合玩家利益（46.46%）**: 包括空投、公售和流动性。
- **团队与顾问（12.81%）**: 合理分配，拥有长期的归属期。
- **开发与运营（11.76%）**: 支持平台增长。
- **投资者（28.97%）**: 为提供资金所必需，但处于少数地位。

<Callout type="success">
**玩家优先**: 近一半的代币通过奖励、空投和可获得的公售分配给玩家，以惠及玩家。
</Callout>

## 归属时间表和锁定期

为确保代币经济稳定并长期协调激励，大多数分配都实施了归属时间表。

### 团队代币（12%）

**归属条款**:
- **总期限**: 4年
- **锁定期**: 1年（无代币释放）
- **释放计划**: 锁定期后按月线性归属
- **理由**: 体现长期承诺

**时间线示例**:
- 第0-12个月：0%释放（锁定期）
- 第13个月：释放2.08%（总数的1/48）
- 第13-48个月：每月释放2.08%
- 第48个月：100%归属

**团队一致性**:
- 激励4年以上的承诺
- 防止早期抛售
- 使团队与长期成功保持一致
- 行业标准做法

### 顾问代币（0.81%）

**归属条款**:
- **总期限**: 2年
- **锁定期**: 6个月
- **释放计划**: 锁定期后按季度释放
- **理由**: 比团队期限短，但仍被锁定

**释放计划**:
- 第0-6个月：0%（锁定期）
- 第9个月：释放16.67%（锁定期后第一个季度）
- 按季度释放：每次16.67%
- 第24个月：100%归属

### 投资者代币（Pre-Seed、Seed、Private：共18.75%）

**归属条款因轮次而异**:

**Pre-Seed（3.28%）**:
- 6个月锁定期
- 2年线性归属

**Seed（6.09%）**:
- 3个月锁定期
- 18个月线性归属

**Private（9.38%）**:
- 3个月锁定期
- 12个月线性归属

**理由**: 早期投资者锁定期更长，以奖励承担风险

### 合作伙伴代币（2.19%）

**定制归属**:
- 按合作伙伴关系协商确定
- 通常为12-24个月
- 可能基于绩效解锁
- 注重战略一致性

### 社区和质押奖励（33.33%）

**逐步释放**:
- **期限**: 5-7年
- **计划**: 年度金额递减
  - 第1年：分配额的10%（3330万代币）
  - 第2年：15%（5000万代币）
  - 第3年：20%（6660万代币）
  - 第4年：20%（6660万代币）
  - 第5年：15%（5000万代币）
  - 第6-7年：剩余20%（6670万代币）

**动态调整**:
- 奖励根据平台活动规模调整
- 如果用户增长超过目标，可以加速
- 如有必要，为了可持续性可以减速
- 由社区投票决定

<Callout type="warning">
**长期分配**: 社区奖励在5-7年内释放，确保代币在平台增长过程中保持稀缺性和价值。
</Callout>

### 金库释放（Reserve：7.5%，R&D：2.06%，Operations：2.20%）

**由治理控制**:
- **需批准**: 代币持有者投票决定重大支出（>金库的1%）
- **透明**: 所有支出公开追踪
- **问责制**: 每季度向社区报告
- **应急储备**: 部分用于关键情况

**用途**:
- 开发资金
- 营销活动
- 战略收购
- 法律与合规
- 应急流动性

## 治理与社区提案

代币持有者通过链上投票机制参与治理。

### 提案流程

**第1步：想法讨论**（7天）：
- 在治理论坛上发布想法
- 社区反馈与完善
- 评估兴趣和支持度
- 非正式投票

**第2步：正式提案**（3天）：
- 在链上提交详细提案
- 要求：持有10,000代币 或 金级质押者
- 包括：规格、理由、实施计划
- 社区审查期

**第3步：投票期**（7-14天）：
- 链上投票开始
- 代币持有者根据持币量和质押倍数加权投票
- 适用法定人数和批准阈值
- 公开结果追踪

**第4步：实施**（可变）：
- 如果通过：开发团队实施
- 时间线取决于复杂性
- 向社区更新进度
- 可能进行最终验证投票

### 提案类别

**游戏功能提案**:
- 新的麻将规则集或变体
- 游戏机制调整
- 锦标赛赛制
- 功能优先级排序
- UI/UX 更改

**经济提案**:
- 市场交易费调整
- 质押奖励修改
- 代币效用扩展
- 销毁机制更改
- 收入分配

**分配提案**:
- 营销预算批准（>10万美元）
- 合作伙伴投资
- 社区计划
- 慈善捐款
- 开发优先事项

**平台治理**:
- 规则和政策变更
- 审核指南
- 服务条款更新
- 战略方向
- 重大平台决策

<Callout type="info">
**民主控制**: 重大平台决策需要社区批准，确保Mahjong Stars根据玩家的兴趣发展。
</Callout>

### 投票要求

**法定人数和批准阈值**:

| 提案类型 | 所需法定人数 | 批准阈值 |
|---|---|---|
| **小幅更改** | 占流通供应量的5% | 51%多数 |
| **重大更改** | 占流通供应量的10% | 66%超级多数 |
| **关键更改** | 占流通供应量的20% | 75%超级多数 |
| **章程更改** | 占流通供应量的30% | 90%超级多数 |

**投票权计算**:
```
基础票数 = 持币量
质押倍数：
  - 无质押：1倍
  - 青铜/白银：1倍
  - 黄金：1.5倍
  - 白金：2倍
  - 钻石：3倍
长期持有者奖励：
  - 6-12个月：+10%
  - 12-24个月：+25%
  - 24个月以上：+50%

总投票权 = 基础票数 × 质押倍数 × (1 + 持有者奖励)
```

**示例**:
- 持有10,000代币
- 白金级质押者（2倍倍数）
- 18个月持有者（+25%奖励）
- 投票权 = 10,000 × 2 × 1.25 = 25,000票

### 委托

**投票权委托功能**:
- 将投票权委托给可信的代表
- 保持投票奖励
- 随时撤销委托
- 可部分委托（例如，委托50%，自行投票50%）
- 可委托给多个代表

**代表系统**:
- 通过持续投票建立声誉
- 透明的投票历史
- 与委托人的沟通
- 社区认可
- 问责机制

## 奖励代币机制

除了治理代币外，还存在一个二次的不可转让奖励系统。

### MStars（奖励货币）

**概述**:
- **不可转让**: 不能发送给其他玩家
- **不可交易**: 不能在市场上出售
- **平台绑定**: 只能在Mahjong Stars内使用
- **充裕**: 设计为可慷慨赚取

**目的**:
- 通过游戏玩法奖励，避免代币风险
- 将游戏经济与投资分开
- 减少监管担忧
- 提供额外的进度层

**赚取MStars**:
- 完成每日任务（100-1,000 MStars）
- 完成每周挑战（1,000-10,000 MStars）
- 参与比赛（每场比赛10-100 MStars）
- 成就（500-50,000 MStars）
- 参与活动（数量不等）

**使用MStars**:
- 兑换成软货币（固定汇率）
- 购买特殊活动入场券
- 解锁装饰品
- 加速角色升级
- 特殊抽奖入场券

**兑换率**:
- 1,000 MStars = 1,000 软货币（1:1）
- 仅单向兑换
- 无兑换限制
- 即时处理

<Callout type="success">
**对玩家友好**: MStars提供了赚取有价值货币的感觉，而没有加密货币的复杂性或风险。
</Callout>

## 经济安全措施

### 反操纵措施

**防护对象**:
- **巨鲸操纵**: 治理限制巨鲸的投票权
- **拉高出货**: 归属机制防止早期抛售
- **市场操纵**: 流动性深度和做市商
- **机器人攻击**: 大额交易需KYC
- **女巫攻击**: 需要账户验证

### 市场稳定

**稳定机制**:
- 逐步释放代币计划
- 从收入中回购并销毁
- 流动性激励
- 专业做市
- 金库干预（如有必要）

**波动性管理**:
- 深层流动性池
- 多家交易所上市
- 多元化的持有者基础
- 长期持有者激励
- 透明沟通
