Le nom "SWR" vient de `stale-while-revalidate`, une stratégie d'invalidation de cache HTTP popularisée par
[HTTP RFC 5861](https://tools.ietf.org/html/rfc5861). SWR est une stratégie qui consiste d'abord à
retourner les données du cache (périmées), puis à envoyer la requête de récupération (revalidation),
et enfin à obtenir les données à jour.
